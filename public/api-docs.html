📋 Overview

The Makseb Statistique API provides comprehensive endpoints for managing restaurant sales data, ticket records, payment information, and cash register closures (Z records). The API supports real-time updates via Socket.IO and includes advanced filtering and pagination capabilities.

ℹ️ Updated Features: This version includes new Z (Cloture) management endpoints and improved multi-filter logic for better data filtering accuracy.

🎫 Tickets & Orders Endpoints

GET /get-tickets FIXED

Get tickets with advanced filtering (Multi-filter logic corrected)

Query Parameters:

  • idCRM (required) - Store ID
  • date1 (required) - Start date (YYYYMMDD)
  • date2 (required) - End date (YYYYMMDD)
  • page (optional, default: 1) - Page number
  • limit (optional, default: 50) - Records per page
  • search (optional) - Text search (ticket #, customer)
  • paymentMethod (optional) - Filter: CARD, CASH, MEAL_VOUCHER, CHECK, etc.
  • fulfillmentMode (optional) - Filter: Sur Place, À Emporter, Livraison
✅ New in this version: Multiple filters now use AND logic correctly.
Example: paymentMethod=CARD&fulfillmentMode=Takeaway returns CARD payments that are Takeaway (not CARD OR Takeaway)
GET /get-orders

Alias for /get-tickets (same implementation for orders listing)

GET /get-tickets-by-z/:idCRM/:zId NEW

Get tickets for a specific closure period (Z number)

Path Parameters:

  • :idCRM (required) - Store ID
  • :zId (required) - Closure ID (Z number)

Query Parameters:

  • search (optional) - Text search
  • paymentMethod (optional) - Payment filter
  • fulfillmentMode (optional) - Fulfillment filter
  • page (optional, default: 1) - Page number
  • limit (optional, default: 50) - Records per page

Response:

  • data - Array of tickets between closure N and N-1
  • totalCount - Total number of tickets in period
  • page - Current page
  • totalPages - Total number of pages
ℹ️ How it works: Returns all tickets between the selected closure's timestamp and the previous closure's timestamp. Perfect for reconciliation by shift.

🔐 Z (Cloture) Management Endpoints NEW

Cash register closure (Z record) management for shift tracking and financial reconciliation.

POST /z

Create a new closure record

Request Body:

  • IDCloture (number) - Closure ID
  • Date_cloture (date) - Closure date
  • HeureCloture (string) - Closure time HH:MM:SS
  • User (string) - Cashier name
  • RECETTE (number) - Total sales
  • RETRAIT (number) - Withdrawal amount
  • IdCRM (string) - Store ID
GET /z/:idCRM

Get all closures for a store (no filtering)

Path Parameters:

  • :idCRM (required) - Store ID
GET /closures/:idCRM NEW

Get closures with optional date range filtering and pagination

Path Parameters:

  • :idCRM (required) - Store ID

Query Parameters:

  • date1 (optional) - Start date (YYYYMMDD)
  • date2 (optional) - End date (YYYYMMDD)
  • page (optional, default: 1) - Page number
  • limit (optional, default: 50) - Records per page

Response:

  • success - Boolean
  • data - Array of closure records
  • totalCount - Total closures in range
  • totalPages - Total number of pages
  • page - Current page
  • limit - Records per page
✨ Feature: Automatically filters closures by selected date range. Used by frontend to populate closure dropdown scoped to current time period selection.
PUT /z/:idCRM/:zId

Update a closure record

Path Parameters:

  • :idCRM - Store ID
  • :zId - Closure ID

Request Body:

  • RECETTE (optional) - Updated sales amount
  • RETRAIT (optional) - Updated withdrawal
  • User (optional) - Updated cashier name
DELETE /z/:idCRM/:zId

Delete a closure record

Path Parameters:

  • :idCRM - Store ID
  • :zId - Closure ID

🔍 Common Filters & Parameters

Payment Methods

Use these values for paymentMethod parameter:

  • CARD - Credit/Debit Card
  • CASH - Cash
  • MEAL_VOUCHER - Meal Voucher
  • CHECK - Check
  • FIDELITY_POINTS - Fidelity Points
  • STORE_CREDIT - Store Credit
  • CORPORATE_ACCOUNT - Corporate Account

Fulfillment/Consumption Modes

Use these values for fulfillmentMode parameter:

  • Sur Place - Dine-in
  • À Emporter - Takeaway/To-go
  • Livraison - Delivery

Date Format

All dates use YYYYMMDD format:

  • 20260330 = March 30, 2026
  • 20260101 = January 1, 2026
  • 20251215 = December 15, 2025

Time Format (Closures)

Time in closure records uses HH:MM:SS format (24-hour):

  • 23:00:00 = 11 PM
  • 12:30:45 = 12:30:45 PM
  • 00:15:30 = 12:15:30 AM

📝 Usage Examples

Example 1: Get Sales Summary

GET /get-sales-summary?idCRM=9999&date1=20260301&date2=20260330

Returns aggregated sales for store 9999 for March 1-30, 2026

Example 2: Filter Tickets with Multiple Criteria

GET /get-tickets?idCRM=9999&date1=20260315&date2=20260330&paymentMethod=CARD&fulfillmentMode=Takeaway&page=1&limit=25

Returns CARD payments that are Takeaway orders (March 15-30, page 1)

Example 3: Get Closures for Date Range

GET /closures/9999?date1=20260315&date2=20260330&page=1&limit=100

Returns closures (Z records) between March 15-30, 2026

Example 4: Get Tickets with Closure Period

GET /get-tickets-by-z/9999/5?paymentMethod=CASH&page=1&limit=50

Returns CASH tickets from closure Z 5's period

Example 5: Create New Closure

❓ Support & Questions

For issues or questions about the API:

  • Check the backend console logs for detailed operation logs
  • Verify date formats (YYYYMMDD) match expected format
  • Ensure idCRM and zId parameters are correct
  • For pagination, use page starting from 1
  • Payment and fulfillment filters are case-sensitive
